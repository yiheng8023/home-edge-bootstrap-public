# bundle/ — offline payload (breaks the chicken-and-egg)

These files let a fresh router install **without any pre-existing wall**. They are git-ignored by
default (large, arch-specific); to make the kit truly clone-and-go, vendor them intentionally:

```powershell
.\scripts\prepare-bundle.ps1
git add -f bundle/mihomo-linux-arm64 bundle/ShellCrash.tar.gz
git add bundle/MANIFEST.json bundle/SHA256SUMS
```

What to place here for an official Asuswrt-Merlin supported ASUS router after verifying its CPU is
**aarch64 / arm64**:

- `mihomo-linux-arm64` — the decompressed Mihomo (Clash.Meta) kernel binary, arm64.
- `ShellCrash.tar.gz` — an offline snapshot of ShellCrash, so install never needs to reach GitHub.
- `SHA256SUMS` and `MANIFEST.json` — local verification metadata generated by
  `scripts/prepare-bundle.ps1`.

`scripts/verify-bundle.sh` checks the bundle without network access. The Merlin adapter installs from
these payloads only when explicitly run with `BOOTSTRAP_INSTALL_RUNTIME=1`; otherwise it configures
and supervises an existing ShellCrash/Mihomo runtime. Only the **subscription** (node list) remains a
manual credential.
