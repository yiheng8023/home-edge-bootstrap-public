# 安全政策

[English](SECURITY.md)

## 受支持版本

| 版本 | 是否支持 |
| --- | --- |
| `v0.1.x` | 是 |
| 更早版本 | 否 |

安全修复面向受支持发布线准备。需要用户操作时，发布物会同时提供升级说明。

## 报告漏洞

请通过仓库的 [Security Advisory 表单](https://github.com/yiheng8023/home-edge-bootstrap-public/security/advisories/new)
报告疑似漏洞。在维护者获得合理时间调查并协调修复前，不要创建公开 issue 或发布利用细节。

报告应包含受影响版本、影响、复现条件，以及最小、经脱敏且不含访问凭据的概念验证。不得提交订阅 URL、节点列表、
凭据、公网 IP 或原始日志。附加脱敏支持包前，应逐个检查文件并删除仍然存在的身份信息或访问
信息。

维护者的响应目标是在七个自然日内确认收到报告，评估范围与严重性，并在重要信息变化时更新
状态。这些是响应目标，不是付费 support SLA，也不保证解决时间。实际可行且不会危及用户时，
披露时间将与报告者协调。
