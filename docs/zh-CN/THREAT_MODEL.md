# 威胁模型

> 中文版是英文 `docs/THREAT_MODEL.md` 的阅读映射；英文文件是项目真源。

本项目管理家庭网络边缘路由器。安全姿态基于有边界的自动化、最小化凭据暴露、本地验证和明确的人工信任决策。

## 受保护资产

- 路由器管理员访问权。
- SSH key 和 authorized keys。
- Wi-Fi 凭据、WAN/ISP 信息、VPN peer key 和无线法规设置。
- 服务商订阅 URL 和节点列表。
- Mihomo external-controller secret，如已配置。
- 离线运行时 bundle 的完整性。
- 路由器流量路径和客户端可达性。

## 信任边界

| 边界 | 项目立场 |
| --- | --- |
| 本机到路由器 SSH | 需要已有授权的 SSH 身份或密码。脚本不收集、不保存路由器密码。 |
| 路由器局域网 | 假设操作者在可信局域网内。WAN 暴露只做审计提示，不在本项目内自动接管。 |
| 服务商订阅 URL | 按凭据处理。不得进入 Git、聊天、日志和公共转换器，除非明确接受。 |
| 订阅转换器 | localhost/私有局域网默认允许。公共转换器必须设置 `SUBSCRIPTION_ALLOW_REMOTE_CONVERTER=1`。 |
| 离线 bundle | 本地 hash 证明 checkout 完整性。上游真实性由供应链策略处理，不能只靠本地 hash。 |
| ShellCrash/Mihomo 运行时 | 按第三方运行时代码处理。本项目负责本地载荷暂存和验证，但不成为上游维护者。 |
| 改版梅林固件 | 只能作为明确兼容性决策接受。官方 Asuswrt-Merlin 仍是优先支持目标。 |

## 威胁与控制

| 威胁 | 控制 |
| --- | --- |
| 订阅凭据进入 Git | `.gitignore`、不打印 URL 的 helper，以及检查 stdout 脱敏的订阅夹具测试。 |
| 公共转换器看到订阅 URL | 默认阻断远端转换器，必须显式打开 allow flag。 |
| 手动改路由器后状态漂移 | 每次人工介入后重新运行只读审计和状态引导循环。 |
| 错误假设 Mihomo API 端口 | 从配置和常见本地端口发现 API，状态输出实际端口。 |
| 服务商/UI 标签变化 | 路线选择基于 Mihomo API 状态、角色化 group 匹配、可选候选/地区表达式、延迟探测和夹具测试。 |
| 路线反复抖动切换 | 自愈切换日志和每小时熔断上限。 |
| 敏感操作出口身份漂移 | 路线健康不能作为身份稳定证明。当前版本只记录负面契约；未来有时限的租约必须绑定运行时权威、有效叶节点、观测出口/网络分类、DNS 路径、策略代次和过期时间，并在权威不清或发生漂移时失效。 |
| 坏部署覆盖可用 kit | 远端 staging 使用临时目录，并保留上一版部署目录。 |
| 已部署脚本静默偏离已审查源码 | 每次应用部署携带源码身份与受管文件哈希；只读路由器状态会验证安装目录和活跃脚本副本。 |
| 错误更新或运行时替换 | 回滚脚本可恢复上一版 kit，并可选择恢复已备份运行时。 |
| 订阅响应异常 | 写入缓存/应用前验证大小、HTML/错误页、raw/base64 和 YAML 形态。 |
| 宿主机环境不清楚 | 本地验证检查必要命令，并运行跨平台夹具测试。 |

## 非目标

- 本项目不绕过刷固件、购买服务商、账号设置、首次路由器凭据、地区无线法规或订阅转换器信任决策。
- 本项目不让公共转换器变安全，只默认阻断并让决策显式化。
- 本项目不认证第三方固件或运行时代码，只记录来源、本地完整性，并保留回滚能力。
- 本项目不保证账号、支付、注册、金融或地区验证结果，也不把国家标签或可达的低延迟路线视为稳定身份。
- 本项目不让已被攻陷的路由器重新可信。如怀疑被攻陷，应先从可信固件和凭据重建。

## 操作规则

1. 假设仓库能在无代理状态恢复新运行时前，先运行 no-wall readiness。
2. 每次手动修改路由器设置后，都重新运行路由器审计。
3. `SUBSCRIPTION.local`、节点列表、key 和含 secret 的日志不得进入 Git 或聊天。
4. 部署、订阅刷新、启用 live self-heal 或回滚前，先使用 dry-run。
5. 公共转换器、固件镜像或第三方运行时替换都必须视为人工信任决策。
