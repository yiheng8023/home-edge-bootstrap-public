# 兼容性政策

[English](../COMPATIBILITY.md) · [首页](../../README.zh-CN.md)

兼容性根据能力与证据声明，不会仅凭品牌、型号系列、适配器成熟度或 fixture 成功来推断。

## 当前已实现参考适配器

已实现 Merlin 适配器面向运行官方 Asuswrt-Merlin 的华硕网关，要求持久化 `/jffs` 存储、
SSH 访问、必需 POSIX 工具，以及受支持的 CPU/运行时组合。运行时路径使用
ShellCrash/ShellClash 与 Mihomo 兼容运行时。

该实现是项目参考适配器。参考角色只说明它是首个已实现架构路径，不代表已经取得“已验证”
成熟阶段，也不会把支持范围扩展到每个华硕型号、Merlin 构建、运行时版本、服务商或网络。

## 两类独立分类

适配器成熟度描述实现责任与证据：

- 外部集成；
- 实验性适配器；
- 社区维护适配器；或
- 已验证适配器。

目标支持分类描述适配器对某一目标的判断：

- **`supported`**：必需能力与适用证据门禁均已满足；
- **`supported_needs_manual`**：目标适合，但仍须完成明确的人工动作；
- **`accepted_modified`**：操作者明确接受兼容改版基线及其来源/支持风险；
- **`unknown`**：证据不足，无法判断固件、架构、存储、shell、运行时或其他必需能力；
- **`unsupported`**：必需能力缺失，或与适配器边界冲突。

适配器不会因成熟度标签自动取得更宽目标支持声明；某一目标的分类也不会提升适配器成熟度。

## 公开证据边界

准确的机器可读要求与分类见
[`config/compatibility-matrix.json`](../../config/compatibility-matrix.json)。其中
`field_evidence` 数组有意保持为空，因此其中列出的目标路径保持为 `unknown`。fixture 不是
现场认证：它们只展示脚本面对合成输入时的
离线行为，不代表某个具体路由器、固件构建、服务商或网络的实测结果。

应用前应运行引导式能力检查并审查每一项不匹配。声明的支持级别不保证安全性、可用性或成功。
软路由或端点回退必须保持独立；回退路径成功不能验证路由器路径。

## 未来适配器

新适配器在接纳前应声明能力契约、维护责任、目标支持政策、fixture、备份与回滚行为、不含敏感
信息的诊断、许可边界和双语操作说明。详见[架构](../ARCHITECTURE.md)与[治理](../../GOVERNANCE.zh-CN.md)。
