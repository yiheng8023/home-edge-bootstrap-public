# Security Policy

[简体中文](SECURITY.zh-CN.md)

## Supported versions

| Version | Supported |
| --- | --- |
| `v0.1.x` | Yes |
| Earlier versions | No |

Security fixes are prepared for the supported release line. Upgrade guidance will accompany a release when user action is required.

## Reporting a vulnerability

Report suspected vulnerabilities through the repository's [Security Advisory form](https://github.com/yiheng8023/home-edge-bootstrap-public/security/advisories/new). Do not open a public issue or publish exploit details before maintainers have had a reasonable opportunity to investigate and coordinate a fix.

Include the affected version, impact, reproduction conditions, and a minimal, redacted proof of concept that contains no access credentials. Do not submit subscription URLs, node lists, credentials, public IPs, or raw logs. Review every redacted support bundle before attaching it and remove any identifying or access-bearing data that remains.

Maintainers aim to acknowledge a report within seven calendar days, assess scope and severity, and provide status updates when material information changes. These are response targets, not a paid support SLA or a guarantee of resolution time. Disclosure timing will be coordinated with the reporter when practical and safe for users.
