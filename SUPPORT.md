# Support

[简体中文](SUPPORT.zh-CN.md)

Choose the channel that matches the request and keep all shared material free of secrets and sensitive network data.

## Questions

For usage or design questions, search the documentation and existing issues first. If the answer is not present, open an issue that clearly states the version, goal, environment class, and where the documentation was unclear. Do not include live network data.

## Bugs

Use the bug report form for reproducible defects. Include minimal synthetic reproduction steps, expected behavior, actual behavior, and the checks you already ran. Before attaching a redacted support bundle, review every file in it yourself.

## Security reports

Do not report vulnerabilities in a public issue. Follow [SECURITY.md](SECURITY.md) and use the repository's Security Advisory form for confidential coordination.

Community support is provided on a best-effort basis. Sponsorship does not create a support SLA, warranty, response priority, or entitlement to a feature.
